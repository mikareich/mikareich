Ganz bestimmt hast du schonmal von dem Begriff API gehört. Jeder Entwickler
stolpert in seiner Laufbahn früher oder später über diesen Begriff, den er ist
in der Informatik allgegenwärtig. Dabei geht es grob gesagt um eine Art
Vermittlung zwischen zwei unterschiedlichen Entwicklungsumgebungen, die
miteinander kommunizieren wollen. Möchte eine Webseite beispielsweise Daten
einer Datenbank in einer anderen Umgebung abrufen, wird oft eine REST-API
verwendet. In diesem Tutorial werden wir selber eine solche REST-API
erstellen, um Flachwitze zu verwalten. Du solltest bereits Erfahrung erste
Erfahrungen mit Node.js und TypeScript mitbringen.

## Was ist eine REST-API?

Eine REST-API (Representational State Transfer API) ist eine Schnittstelle,
die es ermöglicht, Daten zwischen zwei verschiedenen Umgebungen auszutauschen.
Dabei wird das HTTP-Protokoll verwendet, um Daten zu übertragen. Diese Art von
API findet fast überall Anwendung, von JavaScript-Scripts bis hin zu
C++-Code. Dabei wird in fünf verschiedenen HTTP-Methoden unterschieden,
die auch CRUD-Operationen genannt werden:

1. Create (POST): Erstellt eine neue Ressource.
2. Read (GET): Liest eine Ressource aus.
3. Update (PUT, PATCH): Aktualisiert eine Ressource.
4. Delete (DELETE): Löscht eine Ressource.

Bevor wir mit der Implementierung beginnen, empfehle ich dir noch,
[Insomnia](https://insomnia.rest/) oder [Postman](https://www.postman.com/)
zu installieren. Diese Tools helfen dir dabei, deine API zu testen.
